PROBLEM:
--------
When creating a new account, users saw data from previous accounts
in the dashboard (inventory, orders, customers, etc.)

ROOT CAUSE:
-----------
Multiple API endpoints were loading data from ALL profiles instead
of scoping to the active user's profile.

WHAT I FIXED:
-------------
✅ Business Config API - No longer loads random profiles
✅ Dashboard API - Now scoped to active profile_id
✅ Profile Cookie API - Removed dangerous fallbacks
✅ Debug API - Fixed profile loading
✅ Admin Layout - Fixed redirect logic
✅ Login Page - Fixed setup check

FILES CHANGED:
--------------
1. src/app/api/business-config/route.ts
2. src/app/api/dashboard/route.ts
3. src/app/api/set-profile-cookie/route.ts
4. src/app/api/debug-config/route.ts
5. src/app/admin/layout.tsx
6. src/app/admin/login/page.tsx

RESULT:
-------
✅ New accounts see ONLY their own data (fresh dashboard)
✅ Existing accounts see ONLY their own data
✅ Complete multi-tenancy isolation
✅ No data leakage between profiles

HOW TO TEST:
------------
1. Create new account at http://localhost:3000/admin/login
2. Complete setup wizard
3. Dashboard should show ZERO data (fresh account)
4. Should NOT see any data from other accounts

FOR EXISTING USERS:
-------------------
- Log out and log back in to refresh cookies
- Or visit http://localhost:3000/fix-cookie
